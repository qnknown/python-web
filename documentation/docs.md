# Проєкт: Інтернет-магазин харчової продукції з доставкою (B2C та B2B)

## 1. Дослідження предметної області

### Існуючі рішення
На ринку існує багато інтернет-магазинів харчової продукції: локальні сервіси доставки, онлайн-платформи супермаркетів та B2B-маркетплейси.

Стандартний функціонал таких платформ включає:

- каталог товарів із категоріями;
- зручний пошук;
- оформлення та відстеження замовлень;
- оплата при отриманні;
- різні типи доставки (кур’єрська, поштові служби, оптові поставки);
- особистий кабінет покупця;
- адміністративна панель.

**Для B2B клієнтів також доступно:**

- оптові прайс-листи;
- оформлення великих замовлень;
- можливість інтеграції через API.

---

### Необхідний функціонал
Для реалізації платформи, що працюватиме як з кінцевими клієнтами, так і з B2B-сегментом, необхідно забезпечити:

- гнучку систему ціноутворення (знижки, акції, оптові ціни);
- інтеграції з API служб доставки та відстеженням відправлень;
- можливість вибору типу доставки: кур’єрська, на склад, оптова логістика;
- керування замовленнями (статуси, коментарі, історія змін);
- особисті кабінети з історією покупок, улюбленими товарами.

## 2. Основні сутності предметної області

### Користувачі (Users)
- `user_name` — логін користувача  
- email  
- роль (звичайний користувач / адміністратор)

### Товари (Products)
- `id`
- назва
- категорія
- ціна роздрібна
- ціна оптова
- опис, фото, атрибути

### Категорії (Categories)
- `id`
- назва

### Замовлення (Orders)
- `id`
- користувач
- список товарів
- загальна сума
- тип доставки

### Додаткові сутності

#### Cart (Кошик)
- товари
- кількість
- проміжна сума

#### Delivery (Доставка)
- тип доставки
- вартість
- служба доставки

---

## 3. Основні сценарії використання (Use Cases)

### Для B2C клієнтів
- перегляд каталогу товарів
- додавання товарів у кошик
- оформлення замовлення
- вибір типу доставки
- перегляд історії покупок

### Для B2B клієнтів
- авторизація
- перегляд оптових цін
- формування великого замовлення
- оформлення замовлення з прив’язкою складу

### Для адміністратора
- управління товарами
- управління категоріями
- обробка замовлень

---

## 4. Етапи реалізації проєкту

### Етап 1: Базовий функціонал
- каталог товарів
- користувачі

### Етап 2: Реалізація доставки
- кошик
- мінімальна версія модуля замовлень
- доставка
- статуси замовлень
- система трекінгу

### Етап 3: Підтримка B2B
- оптові ціни
- оптові замовлення

### Етап 4: Адмін-панель
- управління товарами
- управління замовленнями

### Етап 5: Фіналізація
- оптимізація коду
- налаштування безпеки
- рефакторинг
